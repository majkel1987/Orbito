## ❌ Problem 1: Auth Interceptor Bug (Blok 1.1)

**Opis**: TODO w bloku 0.3 mówił "będzie dodany w 1.1" - nigdy nie został dodany
**Impact**: Wszystkie API calls zwracały 401 Unauthorized
**Root Cause**:
  - client.ts:20-21 miał tylko komentarz TODO
  - Blok 1.1 został oznaczony jako "passes: true" mimo że interceptor nie został zaimplementowany
  - Frontend wyglądał jak działający (hardcoded dane), więc błąd nie został wykryty

**Fix Applied (2025-12-09)**:
```typescript
// src/core/api/client.ts:22-36
axiosInstance.interceptors.request.use(async (config) => {
  if (typeof window !== "undefined") {
    const session = await getSession();
    if (session?.accessToken) {
      config.headers.Authorization = `Bearer ${session.accessToken}`;
    }
  }
  return config;
});
```

**Lesson**: Auth interceptor MUSI być dodany w tym samym bloku co NextAuth setup.
**Checklist Item**: Przed passes:true sprawdź DevTools Network - requesty MUSZĄ mieć Authorization header.

**Opis**: Komponenty pokazywały "0", "$0" zamiast używać hooków API

**Fix Applied (2025-12-09)**:

- clients/page.tsx - dodano useGetApiClients z pełną tabelą i error handling
- Wszystkie dane teraz pochodzą z prawdziwych API calls

**Lesson**: ZAWSZE używać wygenerowanych hooków, NIGDY nie zostawiać hardcoded danych.
**Checklist Item**:

## ❌ Problem 3: Orval Type Generation (Backend OpenAPI Issue)

**Opis**: Orval wygenerował `customInstance<void>` zamiast `customInstance<ClientDtoPaginatedList>`
**Impact**: TypeScript errors, konieczność type assertions
**Root Cause**: Backend OpenAPI spec nie definiuje response type dla GET /api/Clients
**Workaround Applied**:
```typescript
const { data } = useGetApiClients() as {
  data: ClientDtoPaginatedList | undefined;
  isLoading: boolean;
};
```

**Lesson**: Backend OpenAPI spec MUSI definiować typy response dla wszystkich endpointów.
**TODO Backend**: Dodać ProducesResponseType attributes do kontrolerów.
